# Contributing to AYUSHCare

Thank you for your interest in contributing to AYUSHCare! Here are the professional guidelines for local development.

## Development Workflow

1. **Fork the Repository**
2. **Setup Local Environment**
   - Copy `.env.example` to `.env`
   - Run `npm install` for both frontend and backend
3. **Branching Strategy**
   - Use `feature/...` for new functionality
   - Use `fix/...` for bug fixes
   - Use `refactor/...` for code improvements
4. **Commit Conventions**
   - We follow [Conventional Commits](https://www.conventionalcommits.org/en/v1.0.0/)
   - Examples: `feat: ...`, `fix: ...`, `docs: ...`
5. **Quality Assurance**
   - Run `npm run lint` before submitting PRs
   - Ensure all API endpoints are documented in Swagger
